## 三、当前完整架构（本轮收口后）

```text
UI/ViewModel
   -> Center (Schedule/Sync/Backup/Reminder/Runtime/...)
      -> Core API (operation/query contracts)
         -> StoreDispatcher (直接实现核心 API)
            -> StoreRootNode
               -> Store*Node 子模块
                  -> Room + JSON + CalendarSyncManager(V2)
                     -> System Calendar
```

课程与同步主链：

```text
Course UI Action
  -> MainViewModel
    -> ScheduleCenter
      -> ScheduleOperationApi
        -> StoreDispatcher
          -> StoreRootNode
            -> recurring master/instance/excluded
              -> CalendarSyncManagerV2.syncRecurringMasters
                -> 系统日历
```

## 五、Claude 网页端接手说明（同步/课程）

### 1) 先看这些文件

- 课程映射与兼容：`core/course/CourseEventMapper.kt`
- 课程写操作入口：`ui/viewmodel/MainViewModel.kt`
- 存储编排：`data/node/store/StoreRootNode.kt`
- recurring 落库：`data/migration/LegacyRecurringMigrator.kt`
- 同步主入口：`core/calendar/CalendarSyncManagerV2.kt`

### 2) 当前语义（必须保持）

- 更新全部：修改重复父事件
- 更新本次：分离实例并更新该实例
- 删除本次：排除实例（或删除已分离实例）
- 课程与普通重复日程共享同一同步通道

### 3) 建议回归清单（P0）

1. 新增课程 -> 系统日历是否生成完整学期重复规则
2. 更新全部 -> 未来实例是否整体更新
3. 更新本次 -> 仅该次变化，父系列不被污染
4. 删除本次 -> 仅该次消失
5. 删除整门课 -> 父系列与分离实例是否全部清理
6. 冷启动后课表展示与编辑一致性
