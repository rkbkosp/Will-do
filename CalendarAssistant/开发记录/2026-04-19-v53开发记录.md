## 二、本次核心改动

### 1) 悬浮窗/侧边栏稳定性修复

**问题**

- 侧边栏偶发无法呼出悬浮窗
- 识屏后悬浮窗只隐藏未关闭，后续无法再次正常拉起

**修复**

- `EdgeBarService` 增加手势排除区更新，提升侧滑命中
- `EdgeBarService.startFloatingSchedule()` 增加失败兜底，避免隐藏态卡死
- `FloatingScheduleService`：
  - 常规启动补 `showFloatingWindow()`
  - 识屏成功后调用 `requestClose()`，确保状态可重置

**涉及文件**

- `app/src/main/java/com/antgskds/calendarassistant/service/floating/EdgeBarService.kt`
- `app/src/main/java/com/antgskds/calendarassistant/service/floating/FloatingScheduleService.kt`

---

### 2) 短信采集链路对齐与可用性修复

**目标**

- 对齐成熟方案的“监听与提取”，但后续继续走本项目既有事件入库/提醒链路

**改动要点**

- 注册系统短信通知监听服务（NotificationListener）作为 ROM 兜底通道
- App 启动与开机恢复时执行 listener rebind
- 通知链路加入延迟处理与内容去重
- 去掉 `106` 号段硬过滤，改为“先收后判”
- `SmsContentObserver` 增加持久化游标 + 补扫/轮询兜底，减少漏识别
- `SmsAnalysis` 补充“请用/请凭 XXXX”类取件码识别（覆盖京东样式）

**稳定性修复**

- 排查并修复 `SmsNotificationListenerService` 前台化导致的连接不稳定问题（Android 16 下异常）

**涉及文件（节选）**

- `app/src/main/AndroidManifest.xml`
- `app/src/main/java/com/antgskds/calendarassistant/service/receiver/SmsNotificationListenerService.kt`
- `app/src/main/java/com/antgskds/calendarassistant/service/receiver/SmsReceiver.kt`
- `app/src/main/java/com/antgskds/calendarassistant/core/sms/SmsContentObserver.kt`
- `app/src/main/java/com/antgskds/calendarassistant/core/sms/SmsAnalysis.kt`
- `app/src/main/java/com/antgskds/calendarassistant/App.kt`
- `app/src/main/java/com/antgskds/calendarassistant/service/receiver/BootReceiver.kt`

### 3) 模型配置页交互重构

**本次落地**

- 参数顺序统一为：服务提供商 -> API 地址 -> API Key -> 模型名称

- OpenAI/Gemini 预设模型更新
- 历史默认模型 `gpt-3.5-turbo` 清理，避免误导默认态

### 4) 底栏编辑功能（v53 新增）

**需求落地**
