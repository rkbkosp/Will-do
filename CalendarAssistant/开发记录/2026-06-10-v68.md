## 二、桌面小组件类型扩展

### 2.1 新增天气和课表小组件

在原有日程小组件基础上，本轮补齐了天气小组件和课表小组件：

- `WeatherWidgetProvider` / `WeatherWidgetRenderer`
- `CourseWidgetProvider` / `CourseWidgetRenderer`
- `CourseWidgetSnapshotBuilder`
- `weather_widget_info.xml`
- `course_widget_info.xml`

三个小组件类型统一接入 `WidgetCenter`，由中心负责数据快照构建、实例配置读取和刷新调度。

### 2.2 小组件实例配置

新增 `WidgetInstanceConfigStore`，支持按 widget 实例保存：

- 小组件类型。
- 主题模式。
- 背景透明度。
- 课表当前分段：上午、下午、晚上。

同时补充按类型默认样式能力，让软件内的小组件设置页可以为日程、天气、课表分别保存默认外观。

### 2.3 长按编辑入口

新增 `WidgetConfigureActivity` 及三个类型入口：

- `ScheduleWidgetConfigureActivity`
- `WeatherWidgetConfigureActivity`
- `CourseWidgetConfigureActivity`

系统桌面支持 reconfigurable widget 时，用户仍可通过桌面长按编辑当前小组件实例。该入口只编辑当前实例，不提供类型切换。

## 三、天气小组件展示优化

### 3.1 2x2 预警胶囊修复

旧版 `2x2` 天气小组件在无预警时会显示重复的天气/温度信息，视觉上像无意义胶囊。

本轮改为：

- 有官方预警时显示官方预警短文案。
- 有本地天气风险时显示风险短文案。
- 无官方预警、无风险提醒时隐藏胶囊。

短文案示例：

```text
黄色暴雨预警
2小时后下雨
即将雷雨
```

### 3.2 4x2 / 4x4 天气布局

天气 `4x2` 和 `4x4` 调整为更接近手机系统天气小组件的结构：

- 当前天气摘要。
- 未来 6 小时预报。
- `4x4` 额外展示 5 日预报。

当前没有新增 AQI 等外部接口，优先复用已有天气数据和本地风险分析结果。

## 四、课表小组件修复与展示优化

### 4.1 分段切换崩溃修复

课表 `4x4` 小组件支持上午、下午、晚上分段切换。

旧实现中，分段切换广播会在主线程间接读取 Room 数据，容易触发 Room 的主线程访问异常。

本轮将分段列表构建拆到：

```kotlin
CourseWidgetSnapshotBuilder.buildSections(settings)
```

切换分段时只依赖作息配置，不再主线程查询课程事件，从而避免点击切换按钮崩溃。

### 4.2 空状态误显示修复

课表网格曾出现“明明有课程，但中间仍显示本时段暂无课程”的问题。

根因是空状态判断和实际绘制课程使用了不同过滤路径，叠加 RemoteViews 复用后容易出现状态残留。

本轮改为：

- `renderGrid()` 先统一计算 `sectionItems`。
- `drawCourseBoard()` 和空状态显隐共用同一份 `sectionItems`。
- 隐藏空状态时同时清空文本，降低复用残留风险。

### 4.3 课表网格字体拉伸修复

`4x4` 课表网格由 bitmap 绘制，再放入 `ImageView`。

旧布局使用 `scaleType="fitXY"`，当桌面分配的小组件区域比例和 bitmap 比例不一致时，会非等比拉伸整张图，导致字体被拉长。

本轮调整为：

- `widget_course_board` 使用 `fitCenter`。
- 课表 bitmap 比例调整为更贴近实际显示区域。
- 窄列课程块优先显示课程名，空间不足时不强塞教室信息。

### 4.4 4x2 右侧内容改为汇总胶囊

日程/课表 `4x2` 右侧原本可能出现：

```text
日期 1
卡片 1
日期 2
卡片 2
```

在 ColorOS 等桌面上，这种四段高度容易导致第二张卡片底部被裁切。

本轮改为：

- 左侧保留今日主卡片。
- 右侧只显示一个日期标签和一张主卡片。
- 右下角用小胶囊汇总今明两天内除已展示外的剩余数量。

文案示例：

```text
其他 3 个日程
其他 2 门课
```

汇总数量只统计今天和明天，避免把很远未来的日程全部纳入导致数字异常。

## 五、小组件视觉和透明度修复

### 5.1 背景透明度修复

小组件动态背景本身已经按透明度绘制，但根布局使用的裁剪背景 `widget_background_clip` 是不透明白底，导致用户设置透明度后仍看到白底。

本轮将：

- `res/drawable/widget_background_clip.xml`
- `res/drawable-v31/widget_background_clip.xml`

的 solid 改为透明色，仅保留圆角裁剪效果，使透明度能真实透出桌面壁纸。

### 5.2 颜色标签圆角修复

日程/课表卡片左侧颜色标签原本 XML 使用圆角 drawable，但 renderer 通过：

```kotlin
setBackgroundColor(...)
```

动态改色时会覆盖 shape 背景，导致色条运行时变成矩形。

本轮将所有小组件色条从 `TextView` 改为 `ImageView`，并由 renderer 设置圆角 bitmap：

- `ScheduleWidgetRenderer.bindEventStrip()`
- `CourseWidgetRenderer.bindEventStrip()`

这样 `2x1`、`2x2`、`4x2`、`4x4` 中的日程/课表色条都会保留圆角。

## 六、软件内小组件设置页

### 6.1 类型切换与预览

软件内“桌面小组件”设置页新增类型切换：

```text
日程 / 天气 / 课表
```

切换后显示对应类型的预览和默认样式设置。

默认样式包括：

- 主题模式。
- 背景透明度。

保存后刷新同类小组件。

### 6.2 移除已添加小组件列表

此前曾在软件内设置页底部显示已添加的小组件实例列表。实测该板块会让页面显得复杂，且长按编辑当前实例的系统入口仍保留。

本轮移除该板块，仅保留：

- 类型切换。
- 对应预览。
- 当前类型默认样式编辑。

## 七、课表入口开关与设置整理

### 7.1 新增课表功能开关

新增设置字段：

```kotlin
courseFeatureEnabled: Boolean = true
```

设置链路接入：

- `MySettings`
- `SettingsTransformApi`
- `LocalSettingsTransformApi`
- `SettingsViewModel.updatePreference()`

关闭后只影响两个入口：

1. 首页不能下滑展开课表。
2. 点击课表小组件只回首页，不展开课表。

课程数据、课程管理、作息设置、课表小组件显示和课表分段切换都不受影响。

### 7.2 课表小组件点击跳转

新增 widget action：

```kotlin
WidgetActions.ACTION_OPEN_COURSE
```

课表小组件根布局点击后会打开 App：

- 如果课表功能开启，进入首页并展开课表。
- 如果课表功能关闭，只进入首页。

为避免一次 widget 请求在返回首页时重复触发展开，`HomePage` 中记录 `lastHandledOpenCourseRequestId`，确保同一个请求只消费一次。

### 7.3 偏好设置新增课表板块

`偏好设置` 新增独立“课表”板块，包含：

- `启用课表功能`
- `学期配置`
- `管理所有课程`
- `作息时间设置`

其中三个跳转项会跟随开关展开/收起，动画复用设置页现有风格：

```kotlin
expandVertically() + fadeIn()
shrinkVertically() + fadeOut()
```

`学期配置页` 中原有的课程管理和作息时间设置入口保留，不做删除。

### 7.4 主页提示调整

主页今日空状态移除“下滑以打开课表”提示，改为更中性的：

```text
今日暂无日程
```

搜索无结果时仍显示：

```text
未找到相关日程
```
