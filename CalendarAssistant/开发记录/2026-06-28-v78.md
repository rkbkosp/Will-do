### 2.2 识别模式链路

补回 v75 识别模式入口：

- `AI_ONLY`
- `REGEX_ONLY`
- `REGEX_THEN_AI_ON_EMPTY`
- `REGEX_THEN_AI_REVIEW`

主要恢复点：

- `RecognitionTextNode` 重新按 `RecognitionModePolicy` 路由文本识别。
- `REGEX_THEN_AI_ON_EMPTY` 在正则无结果时回退 AI。
- `REGEX_THEN_AI_REVIEW` 先用正则结果入库，再由后置复核链路触发 AI 复核。
- `RecognitionTextNode.analyzeTextEventsByAi(...)` 保持强制 AI 路径，避免 AI 复核时递归尊重当前识别模式。

### 2.3 正则 AI 复核链路

补回后置复核上下文：

- `RecognitionCompletedEvent` 增加历史字段 `rawText`。
- `RecognitionCenter` 在文本识别成功事件中写入原始文本。
- `ContentIngestCenter` 在 review 模式、单候选、rawText 非空时构造 `RegexAiReviewCoordinator.ReviewContext`。
- 入库成功且只创建一条日程时，调用 `RegexAiReviewCoordinator.reviewAfterIngest(...)`。
- `App` 中恢复 `RegexAiReviewCoordinator` 的懒加载装配，并传给 `ContentIngestCenter`。

本轮没有重新引入已撤回的自设计字段，例如 `recognitionSettings`、`regexReviewRequested`。

## 四、patch 对比后补回的问题

### 4.1 随口记备份链路

patch 对比发现备份页迁移过程中，随口记备份相关选项和结果展示被误删。本轮已补回完整链路，不只恢复 UI。

补回内容：

- `AppBackupOptions.includeQuickMemos`
- `AppBackupData.quickMemos`
- `AppBackupData.quickMemoSuggestions`
- `AppBackupQuickMemoDto`
- `AppBackupQuickMemoSuggestionDto`
- `AppBackupImportResult.quickMemosImported`
- `QuickMemoDao.getAllQuickMemos()`
- `QuickMemoDao.getAllSuggestions()`

`BackupCenter` 补回：

- 导出 JSON 时禁止包含附件或随口记，因为随口记录音需要 ZIP。
- ZIP 导出可同时包含附件和随口记录音。
- 随口记录音写入 `quick_memos/audio/`。
- ZIP 导入时提取 `quick_memos/audio/` 到临时目录。
- 导入随口记正文、状态、排序、待办状态和建议。
- 录音文件导入到 App 私有目录 `quick_memos/audio`。
- 用基础重复键跳过重复随口记。

备份页补回：

- “数据备份”文案包含“随口记”。
- 导出说明改为“包含附件或随口记时会导出 ZIP”。
- 导出选项增加“随口记”。
- 导入 ZIP 时默认勾选随口记。
- 导入结果显示“随口记 N 条”。

### 4.2 每日提醒开发测试入口

patch 对比发现开发测试弹层中的每日提醒测试入口缺失。本轮已补回：

- `今日提醒`
- `明日预告`

入口位置：

```text
LaboratoryPage.kt → 通知/胶囊测试 → 每日提醒
```

实现方式：

- 优先用 `dailySummaryQueryApi.buildPayload(...)` 基于当前设置、日程和天气生成 payload。
- 无有效 payload 时使用开发者兜底 payload。
- 兜底内容复用 `ScheduleNormalDisplay.dailySummaryTitle(...)` 与 `dailySummaryContent(...)`。

本轮按用户要求，没有恢复“本地录音测试”入口。

### 4.3 音量+长按动作兜底

patch 对比发现 `PreferenceSettingsPage` 中开启“长按音量+”时，原有动作值兜底被简化。本轮已补回：

```kotlin
volumeUpLongPressAction = if (isChecked) {
    settings.volumeUpLongPressAction.coerceIn(1, 3)
} else {
    settings.volumeUpLongPressAction
}
```

这样即使历史设置中保存了异常动作值，重新开启时也会回到合法范围。
